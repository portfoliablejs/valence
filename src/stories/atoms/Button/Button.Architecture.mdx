# `ds-button`

The `ds-button` component is an **atomic, multi-use action primitive** in the Valence Design System. Within the `portfoliable` web application shell, it serves as a core action target across multiple views and composite molecules, such as primary call-to-actions in the Case Reader, external repository links, media controls in `ds-video-player`, and social share triggers.
<br />

## Principles
<br />
### Unidirectional Data Flow & State Synchronization
The component follows a strict **State In / Events Out** lifecycle model:
* **State In (Observed Attributes):** The component observes changes to core state attributes (`variant`, `aria-label`, `disabled`, `icon`, `icon-variant`, `icon-position`). When attributes mutate, `updateAttributes()` updates the internal Shadow DOM button element and the child `<ds-icon>` tag.
* **Events Out (Native Interaction):** User interaction triggers standard `click` events emitted from the host.
* **Encapsulation:** The component handles internal focus styling and ARIA delegation without exposing its inner Shadow DOM subtree.

## Slot Architecture

### Internal Shadow DOM Structure
The Shadow DOM template is intentionally compressed on a single line to prevent unwanted whitespace text nodes around slot projections:

```html
<style>${css}</style>
<button type="button">
  <ds-icon class="btn-icon" style="display: none;"></ds-icon>
  <slot></slot>
</button>
```

### Iconography & Tooltip Synergy
1. **Child `<ds-icon>` Mechanics:** When the `icon` attribute is present, `ds-button` automatically displays the internal `<ds-icon>` component. It calculates icon sizing dynamically: circular variants (`icon` and `floating`) receive `size="20"`, while pill and text variants receive `size="14"`.
2. **Icon Position Reversal:** Setting `icon-position="right"` sets flex direction to `row-reverse` on the internal button container.
3. **Tooltip Composite Pattern:** When paired with `<ds-tooltip>`, hovering or focusing the button activates the tooltip's `visible` attribute to display helper text and keyboard shortcuts.

## `a11y` Engine

### 1. ARIA Delegation & Host Scrubbing
When `aria-label` is applied to `<ds-button>` in HTML, `updateAttributes()` delegates the attribute down to the internal shadow `<button>` element and scrubs `aria-label` from the host tag. This prevents browser screen readers and automated accessibility checkers from reporting `aria-prohibited-attr` on non-interactive custom element wrappers.

### 2. Root Mutation Observer (`_observeRootAccessibility`)
During `connectedCallback()`, `ds-button` instantiates a `MutationObserver` on `document.documentElement`. It reflects root accessibility classes as host boolean attributes:

## Examples

### Example A: Primary CTA Button (Case Reader View)
Used inside the Case Reader to launch the primary video presentation:

```javascript
import 'valence/ds-button';

const container = document.createElement('div');
container.innerHTML = `
  <ds-button 
    variant="primary" 
    icon="play" 
    icon-position="left">
    Play Video
  </ds-button>
`;

const button = container.querySelector('ds-button');
button.addEventListener('click', () => {
  appShell.openVideoPlayer();
});
```

### Example B: Floating Control Button with Keyboard Shortcut (Video Player)
Used inside `ds-video-player` controls as an elevated overlay action:

```javascript
import 'valence/ds-button';
import 'valence/ds-tooltip';

const wrapper = document.createElement('div');
wrapper.style.position = 'relative';

wrapper.innerHTML = `
  <ds-button 
    variant="floating" 
    icon="play" 
    aria-label="Play Video">
  </ds-button>

  <ds-tooltip 
    text="Play Video" 
    position="top" 
    show-kbd 
    kbd-label="Esc">
  </ds-tooltip>
`;

const btn = wrapper.querySelector('ds-button');
const tooltip = wrapper.querySelector('ds-tooltip');

btn.addEventListener('mouseenter', () => tooltip.setAttribute('visible', 'true'));
btn.addEventListener('mouseleave', () => tooltip.removeAttribute('visible'));
```

### Example C: Icon-Only Action (Social Sharing)
Used inside header toolbars and case cards to trigger share links:

```javascript
const shareBtn = document.createElement('ds-button');
shareBtn.setAttribute('variant', 'icon');
shareBtn.setAttribute('icon', 'share');
shareBtn.setAttribute('aria-label', 'Share case study');

shareBtn.addEventListener('click', async () => {
  if (navigator.share) {
    await navigator.share({ title: document.title, url: window.location.href });
  }
});
```
