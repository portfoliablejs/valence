# `ds-slider-dot`

The `ds-slider-dot` component is a **presentational / controlled** Atom designed to render compact pagination dots. It is fully accessible, utilizes native `<button>` primitives inside its Shadow DOM boundaries, and triggers unified custom events when clicked.

---

## Principles

### Unidirectional Data Flow
The component strictly adheres to the **State In / Events Out** pattern:
* **State In (Attributes / Properties):** The host binds attributes (`active`, `aria-label`) to toggle active scales or provide accessible text strings.
* **Events Out (`ds-dot-click`):** Dispatches standard customized events when user interactions click and trigger a slide transition.

## Accessibility & Host ARIA Delegation

* **Button Delegation:** Uses an internal `<button type="button">` to support native keyboard focus paths, spacebar trigger activation, and correct user flows.
* **ARIA Host Scrubbing:** Automatically scrubs standard Light DOM `aria-label` attributes applied to host nodes and delegates them down to the focusable button.
* **Global Access Syncing:** Leverages `_observeRootAccessibility` to track HTML root class adjustments:
  * `.a11y-dark-mode`: Shifts inactive dots to higher contrast gray palettes.
  * `.a11y-high-contrast`: Enforces clear borders and filled circles for AAA compliance.
  * `.a11y-reduce-motion`: Disables spring-scaling transitions.
  * `.a11y-forced-colors`: Overrides backgrounds via system `CanvasText` (inactive) and `Highlight` (active).

## Examples

### Basic Integration Example

```javascript
import 'valence/ds-slider-dot';

const dot = document.createElement('ds-slider-dot');
dot.setAttribute('active', 'true');
dot.setAttribute('aria-label', 'Go to slide 4');

// Clean sub-atomic overrides
dot.style.setProperty('--ds-slider-dot-size', '16px');
dot.style.setProperty('--ds-slider-dot-active-bg', 'var(--color-success)');

dot.addEventListener('ds-dot-click', (e) => {
  console.log('Slide dot clicked! Active state:', e.detail.active);
});

document.body.appendChild(dot);
```
