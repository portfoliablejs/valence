# `ds-check`

The `ds-check` component is a **controlled / presentational** Atom designed to render vector graphic selection indicators with optional surface background containers, built-in keyboard/click toggling, and accessibility synchronization.

---

## Principles

### Unidirectional Data Flow
The component strictly adheres to the **State In / Events Out** pattern:
* **State In (Attributes / Properties):** The host application or parent component binds primitive attributes (`selected`, `has-background`, `disabled`, `aria-label`) to control rendering state.
* **User Interactions:** Click and keyboard events (`Space`, `Enter`) dynamically toggle state and update internal accessibility attributes (`aria-checked`, `aria-disabled`).
* **Encapsulated State:** Visual presentation and SVG animation choreography are encapsulated within the Shadow DOM.

## Slot Architecture

The `ds-check` component is a self-contained visual primitive that encapsulates its vector graphics directly inside its Shadow DOM. It does not expose public slots, ensuring uniform geometric alignment, stroke thickness, and animation keyframes across all instances.

## Accessibility & Host ARIA Delegation

* **Role Delegation:** Automatically initializes `role="checkbox"` on the host custom element.
* **ARIA Attribute Sync:** Real-time updates sync `aria-checked` and `aria-disabled` based on host attribute changes.
* **Keyboard Interaction:** Supports full keyboard navigation with `Tab` focusing (`tabindex="0"`) and trigger handling via `Space`, `Spacebar`, or `Enter` keys.
* **Global Accessibility Propagation:** Integrates `_observeRootAccessibility` using a `MutationObserver` on `document.documentElement` to synchronize global accessibility flags:
  * `.a11y-dark-mode`: Adjusts surface boundary contrast.
  * `.a11y-high-contrast`: Enforces strict binary contrast overrides.
  * `.a11y-forced-colors`: Maps system active `Highlight` and `GrayText` palettes for Windows High Contrast Mode.
  * `.a11y-reduce-motion`: Completely suppresses transition durations and delays for reduced motion preferences.

## Examples

### Basic Integration Example

```javascript
import 'valence/ds-check';

const check = document.createElement('ds-check');
check.setAttribute('selected', '');
check.setAttribute('has-background', '');
check.setAttribute('aria-label', 'Accept terms and conditions');

// Apply sub-atomic style overrides via CSS custom properties
check.style.setProperty('--ds-check-color', 'var(--color-success)');
check.style.setProperty('--ds-check-size', '24px');

check.addEventListener('click', () => {
  console.log('Check toggled. Is selected:', check.hasAttribute('selected'));
});

document.body.appendChild(check);
```
