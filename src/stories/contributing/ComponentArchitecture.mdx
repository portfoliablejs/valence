# Universal Component Architecture Documentation Standard

This document defines the architectural specification and template for authoring component-level architecture files (`[ComponentName].Architecture.mdx`). Every component in the **Valence Design System**—regardless of its atomic tier—must include a dedicated architecture file alongside its `.js`, `.css`, and `.stories.js` files.

---

## Architectural Blueprint

Architecture documentation bridges the gap between basic UI controls and full software integration. While standard Storybook stories (`.stories.js`) document visual states and argTypes, the `.Architecture.mdx` file details:

* **Data Ownership & Flow:** Unidirectional data flow contracts (State In / Events Out).
* **Composition & Slots:** Slot hierarchy, child constraints, and sub-component synergy.
* **API Contracts:** Typed data schemas, custom event detail payloads, and CSS property hooks.
* **Accessibility Engine Mechanics:** Host ARIA scrubbing, keyboard focus loops, and root class propagation.
* **Host Application Wiring:** Concrete SPA integration examples.

---

  ## Principles

  ### Unidirectional Data Flow
  The component strictly adheres to the **State In / Events Out** pattern:
  * **State In (Attributes / Properties):** The host application binds primitive attributes or complex properties onto the host element.
  * **Events Out (`ds-[component]-*`):** User interactions trigger custom DOM events containing immutable event detail payloads.
  * **Encapsulated State:** The component never mutates external application state or history directly.

  ## Accessibility & Host ARIA Delegation

  * **ARIA Scrubbing:** Attribute delegation logic strips `aria-*` attributes applied to the outer host custom element tag and delegates them to the inner focusable primitive element.
  * **Keyboard Navigation:** Native keyboard interactions supported (`Tab`, `Enter`, `Space`, `Escape`, Arrow Keys).
  * **Global Accessibility Propagation:** Listens to class changes on `document.documentElement` via `MutationObserver` to sync accessibility modes:
  * `.a11y-dark-mode`: Inverts contrast boundary scales.
  * `.a11y-high-contrast`: Applies 21:1 binary contrast ratios.
  * `.a11y-forced-colors`: Applies system active `Canvas` / `Highlight` palette maps.
  * `.a11y-reduce-motion`: Suppresses CSS animation transitions.
