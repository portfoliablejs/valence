# `ds-audio-player`

The `ds-audio-player` component is an interactive **controlled** Molecule designed to manage audio reading tracks, speed variations, volume settings, and synchronized viewport auto-scrolling. It orchestrates nested custom `ds-button`, `ds-seek-bar`, `ds-divider`, and `ds-tooltip` Atoms, implementing dynamic pointer-drag and keyboard-traversal controls.

---

## Principles

### Unidirectional Data Flow
The component strictly adheres to the **State In / Events Out** pattern:
* **State In (Attributes / Properties):** The host shell binds properties (`playing`, `time`, `duration`, `speed`, `hide-on-scroll`, `auto-scroll`, `volume`, `muted`, `variant`) and customized translation tags (`label-*`) to direct the player layout state.
* **Events Out:** User interactions with buttons or sliders emit clean customized event streams prefixed with `ds-audio-` carrying immutable detail payloads upstream.

### Sub-Atomic CSS Custom Property Hooks

This composite molecule encapsulates nested Atom sub-atomic overrides to align colors and spacing parameters:
- `ds-divider` margin: `--ds-divider-margin` is overridden to `0`.
- `ds-seek-bar` colors: `--ds-seek-bar-fill-bg`, `--ds-seek-bar-knob-bg`, and `--ds-seek-bar-track-bg` are cleanly set on `.volume-seek` elements.

## Accessibility & Host ARIA Delegation

* **Screen Reader Announcement:** Assigns `role="region"` and `aria-label="Audio Player"` on the container, managing aria-live elements for timeline progression.
* **Keyboard Hotkeys:** Focus on the entire component activates arrow-key hotkey navigation (`ArrowRight` / `ArrowLeft` seek relative seconds; `ArrowUp` / `ArrowDown` increments volume parameters). Native button controls support standard focus paths and `Enter` / `Space` executions.
* **Zero i18n Localization:** Fully translates screen-reader announcers, tooltips, and play descriptions by binding `label-[control]` attributes downstream.
* **Global Access Syncing:** MutationObserver on document roots reflects global setting cascades downstream as host element boolean flags:
  * `.a11y-dark-mode`: Shifts background elements to dark gray player HUD tokens.
  * `.a11y-high-contrast`: Enforces strict AAA borders and monochrome layouts.
  * `.a11y-dyslexia`: Forces Comic Sans MS fonts.
  * `.a11y-reduce-motion`: Disables volume expand transitions and snatches layout coordinates instantly.

### Basic Integration Example

```javascript
import 'valence/ds-audio-player';

const player = document.createElement('ds-audio-player');
player.setAttribute('time', '45');
player.setAttribute('duration', '180');
player.setAttribute('volume', '75');

// Binding custom translations
player.setAttribute('label-reader', 'Leitor de ûudio');
player.setAttribute('label-play', 'Iniciar');
player.setAttribute('label-pause', 'Pausar');

player.addEventListener('ds-audio-play-toggle', (e) => {
  console.log('Player toggled. Is playing:', e.detail.playing);
});

document.body.appendChild(player);
```
