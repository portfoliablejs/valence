# `ds-summary`

The `ds-summary` component is a **stateless (presentational)** Molecule designed for rendering summaries, executive briefs, and key case study findings using glassmorphic audio-player container styles and controllable slotted metric grids.

---

## Principles

### Unidirectional Data Flow
The component strictly implements the **Unidirectional Data Flow** pattern:
* **State In (`text`, `active`, `label-header`, `show-metrics`):** The host application binds primitive text and active state attributes downstream onto the host element.
* **Events Out (`ds-summary-complete`):** Emits a custom DOM event when the dynamic typing sequence finishes rendering text and reveals slotted metric cards.
* **Passive Presentation:** The component contains zero business logic and relies on direct attribute bindings for localizable header labels.

## Accessibility & Host ARIA Delegation

* **ARIA Scrubbing:** Purges host-level `aria-label` attributes and delegates them downstream onto the internal Shadow DOM summary container.
* **Live Regions:** Renders `aria-live="polite"` on the summary text container to inform screen readers as dynamic typing updates finish.
* **Global Accessibility Propagation:** Listens to class changes on `document.documentElement` via `MutationObserver` to sync accessibility modes:
  * `.a11y-dark-mode`: Adjusts glass container, border, and text contrast levels.
  * `.a11y-high-contrast`: Enforces solid black background and 2px solid white borders.
  * `.a11y-large-text`: Scales body copy font size to 18px and header to 13px.
  * `.a11y-dyslexia`: Applies high-legibility letter spacing and font stacks.
