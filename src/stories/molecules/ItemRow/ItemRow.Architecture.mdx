# `ds-item-row`

The `ds-item-row` component is an interactive, presentational **controlled** Molecule designed to render uniform, highly accessible list rows, settings items, and navigation blocks. It aggregates custom leading `ds-icon` and trailing `ds-kbd` Atoms on the left/center, while housing configurable form `ds-toggle`, `ds-check`, or `ds-radio` Controls on the right.

---

## Principles

### Unidirectional Data Flow
The component strictly adheres to the **State In / Events Out** pattern:
* **State In (Attributes / Properties):** The host application binds styling variables, keyboard modifiers, and active form states (`label`, `icon`, `show-icon`, `control`, `active`, `selected`, `disabled`) downstream.
* **Events Out (`ds-row-click`):** Captures mouse clicks or spacebar/enter strokes across the container surface, toggling attributes depending on active right-side controls, and dispatches unified event payloads.

## Slot Architecture

### Sub-Atomic CSS Custom Property Hooks

This composite molecule overrides sub-atomic variables on nested components to ensure proportional scaling inside navigation cards:
* `ds-toggle`: overrides track size and knob size using `--ds-toggle-width: 44px`, `--ds-toggle-height: 24px`, and `--ds-toggle-knob-size: 18px`.
* `ds-check`: overrides bounding box size using `--ds-check-size: 20px`.
* `ds-radio`: overrides bounding box size using `--ds-radio-size: 20px`.
* `ds-icon`: overrides leading icon size using `--ds-icon-size: var(--size-icon-md, 20px)`.

## Accessibility & Host ARIA Delegation

* **Aria Roles & Traversal:** When `control="none"`, delegates `role="button"` and `tabindex="0"` to the row container for focus-visible navigation. When form controls are active, the row is treated as a passive grouping, delegating focus directly into nested `ds-toggle`, `ds-check`, or `ds-radio` elements.
* **ARIA Host Scrubbing:** Automatically purges Light DOM `aria-label` tags and applies them downstream directly onto focusable primitives.
* **Global Access Syncing:** MutationObserver on document roots reflects global setting cascades downstream as host element boolean flags:
  * `.a11y-dark-mode`: Adjusts default hover background contrast boundaries.
  * `.a11y-high-contrast`: Enforces strict AAA border separators.
  * `.a11y-dyslexia`: Overrides row label typefaces with Comic Sans MS.
  * `.a11y-reduce-motion`: Forces immediate 0ms hover backgrounds.
  * `.a11y-forced-colors`: Employs Windows active system colors (`Canvas`, `Highlight`, and `HighlightText`).

## Examples

### Basic Integration Example

```javascript
import 'valence/ds-item-row';

const row = document.createElement('ds-item-row');
row.setAttribute('label', 'Enable Subtitles');
row.setAttribute('show-icon', '');
row.setAttribute('icon', 'language');
row.setAttribute('control', 'toggle');
row.setAttribute('active', 'true');
row.setAttribute('toggle-text-on', 'ON');
row.setAttribute('toggle-text-off', 'OFF');

row.addEventListener('ds-row-click', (e) => {
  console.log('Row clicked! Form switch status:', e.detail.active);
});

document.body.appendChild(row);
```
