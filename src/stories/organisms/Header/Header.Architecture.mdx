# `ds-header`

The `ds-header` component is a **stateless (presentational)** Organism designed to compose top-level application navigation chrome. It places a controlled `ds-breadcrumb` region on the far left and a `ds-navigation-menu` region on the far right while preserving a strict **State In / Events Out** contract for the host application.

---

## Principles

### Dumb Composition Boundary
The component strictly adheres to the **Unidirectional Data Flow** paradigm:
* **State In (Attributes / Properties):** The host app binds breadcrumb arrays, visibility flags, and navigation-menu presentational attributes onto `<ds-header>`.
* **Events Out (Child Events Forwarded):** The organism does not create domain state of its own. Instead, it allows child events such as `ds-breadcrumb-select` and `ds-navigation-menu-language-select` to bubble outward unchanged.
* **No Smart App Logic:** Routing, locale changes, accessibility preference persistence, and profile actions remain fully external to the component.

### Fixed Edge Alignment
The visual contract is intentionally rigid:
* **Left Anchor:** The breadcrumb region expands from the far left edge of the header content row.
* **Right Anchor:** The navigation menu remains pinned to the far right edge through flex layout and auto margin separation.
* **Viewport Shell:** The host element and internal shell stretch to the full viewport width, making the organism suitable for app-level framing rather than inline card use.

## Accessibility & Semantics

* **Semantic Landmark:** The organism renders an internal `<header>` element and delegates `aria-label` from the host for top-level navigation context.
* **Child Accessibility Preservation:** Accessibility mechanics remain owned by `ds-breadcrumb` and `ds-navigation-menu`, allowing those molecules to preserve their own tooltip, keyboard, and contextual menu semantics.
* **Visibility Without State Loss:** Optional regions are hidden through presentational toggles rather than destructive mutation of application state, keeping host control explicit and reversible.
