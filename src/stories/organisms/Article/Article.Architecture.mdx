# `ds-article`

The `ds-article` component is a **stateless (presentational)** Organism designed to orchestrate the layout logic for long-form reading experiences. It provides rigid bounds and toggleable structural zones for typography, metadata, action buttons, summaries, media players, and floating navigation docks.

---

## Principles

### Unidirectional Data Flow & Layout Management
The component strictly adheres to the **Unidirectional Data Flow** paradigm:
* **State In (Attributes & Slots):** The host application binds strings for the header/actions and projects complex Web Components (`ds-summary`, `ds-audio-player`, `ds-case-navigator`) through named `<slot>` channels.
* **Events Out (`ds-article-*`):** Action buttons and social share icons emit standardized custom events up to the router/shell to handle link routing and native share API triggers.
* **Typographic Inheritance:** Body copy passed into the default slot relies natively on Light DOM inheritance for `<p>`, `<h3>`, and `<code>` tags mapping directly to global sub-atomic typographic scale definitions.

### Scroll Transition Engine (Thumbnail Retraction)
The reading layout uses a **continuous progress-driven transition** instead of a binary threshold snap:
* **Progress Source:** Scroll position is normalized and eased in JavaScript, then written as `--ds-article-thumbnail-progress`.
* **Smooth Interpolation:** The grid thumbnail column width, inter-column gap, thumbnail opacity/offset, and content column offset are interpolated from that single progress variable.
* **Bidirectional Consistency:** The same interpolation path is used for scroll-down and scroll-up, so expansion and retraction feel symmetrical.
* **Body Stability:** Article body children no longer scale/fade during this transition, preventing perceived paragraph "bouncing" while preserving the thumbnail motion effect.

## Accessibility & Host ARIA Delegation

* **Tooltips & KBD:** Action buttons pair natively with `ds-tooltip` primitives. The primary action tooltip explicitly registers `kbd-label="Enter"` indicating its global keyboard shortcut.
* **Global Accessibility Propagation:** Listens to class changes on `document.documentElement` via `MutationObserver` to sync accessibility modes:
  * `.a11y-dark-mode`: Inverts title typography, mutes secondary labels, and adjusts all `::slotted()` Markdown elements (`<blockquote>`, `<pre>`) to dark backgrounds.
  * `.a11y-high-contrast`: Enforces `#FFFFFF` solid fills and borders on all header text arrays and slotted content.
  * `.a11y-large-text`: Scales the article title by a 1.25x multiplier.
  * `.a11y-dyslexia`: Re-renders the entire article structure using the high-legibility font stack.
