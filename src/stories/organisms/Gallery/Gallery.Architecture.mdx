# `ds-gallery`

The `ds-gallery` organism is a controlled horizontal case-study strip that composes `ds-gallery-item` molecules and keeps drag motion, keyboard focus, and layout orchestration inside the Shadow DOM.

---

## Principles

### Unidirectional Data Flow
The component follows the **State In / Events Out** pattern:
* **State In (Attributes / Properties):** Host applications provide `item-count`, `engine`, an optional `items` array, and the gallery gap through `--gallery-item-gap`.
* **Events Out (`ds-case-select`):** Child gallery items emit selection intent through a bubbling, composed custom event that reaches the gallery host.
* **Derived Presentation State:** Drag offset, focus index, and the `is-dragging` host state are derived entirely from local interaction state.

## Accessibility & Host Integration

* **Keyboard Navigation:** The gallery viewport is focusable and supports Arrow Left / Right, Home, End, Enter, and Space.
* **Pointer Experience:** The viewport uses `touch-action: pan-y` so horizontal drag can be handled by the component without blocking vertical page scrolling.
* **Host State Reflection:** The gallery toggles `is-dragging` on the host and viewport during pointer interaction so the whole component can animate as a unit.
* **Reduced Motion Compliance:** When the root document carries `.a11y-reduce-motion`, the gallery reflects that state to the host, removes scale/transition animations, and uses a non-animated release path.
* **Focus Delegation:** Keyboard selection moves focus into the active `ds-gallery-item` thumbnail surface without leaving the gallery context.

## Integration Example

```javascript
import 'valence/src/stories/organisms/Gallery/Gallery.js';

const gallery = document.createElement('ds-gallery');
gallery.engine = 'minimal';
gallery.itemCount = 5;
gallery.style.setProperty('--gallery-item-gap', '100px');

gallery.addEventListener('ds-case-select', () => {
  console.log('Open the selected case study');
});

document.body.appendChild(gallery);
```
