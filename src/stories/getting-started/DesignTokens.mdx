<Title>Design Tokens</Title>

Design tokens are the foundational, indivisible values of our visual language. They are defined as CSS Custom Properties and are used by all components to ensure consistency. You should also use these tokens in your application for any custom styling.

## Colors

Our color palette is divided into base, grayscale, and semantic colors. To use a color token in your CSS:

```css
.my-custom-element {
  background-color: var(--color-card-bg);
  color: var(--color-accent);
}
```

## Spacing & Radii
We use a consistent 8-point based spacing scale for margins, padding, and layout. Radii are used for rounding corners.

```css
.my-card {
  padding: var(--space-lg); /* 16px */
  margin-bottom: var(--space-xxl); /* 32px */
  border-radius: var(--radius-md); /* 8px */
}
```

## Typography
The typography scale defines the font families, sizes, and weights used throughout the application. While components handle their own typography, you can use these tokens for slotted content or custom elements.
```css
.my-custom-text {
  font-family: var(--font-family-base);
  font-size: var(--font-size-md);
}
````

## Animations
Animation tokens provide consistent timing and easing curves for transitions and motion.

```css
.my-interactive-element {
  transition: transform var(--anim-normal) var(--anim-ease);
}
```
